## Почему вы не можете не использовать API
- String выгоднее использовать когда у нас простой случай, где не нужно менять строки
- StringBuilder нужно использовать когда много раз изменения данных
- когда создаем нашу программу, нужно думать что программа может быть запущена на разных платформах. Лучше использовать относительные пути. В идеале код должен работать без правок на Windows и Linux
- по возможности лучше избегать блоков try, catch
- при построении программы, нужно выделять основной функционал программы для выполнения определенных задач
- предпочтительнее импортировать только нужные библиотеки, вместо *, даже если будет много импортов
- нужно вести логирование всех действий, работы функции, метода, нажатии кнопок, для дальнейшей отладки приложения, в случае возникновения ошибок
