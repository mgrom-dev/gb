# Инструкция для работы с Markdown

## Выделение текста
\**Для выделения текста курсивом, нужно выделить его звездочками или символом нижнего подчеркивания (_)*\*

\*\***Для выделения текста полужирным, нужно выделить его двойными звездочками или двойным символом нижнего подчеркивания(__)**\*\*

\**Альтернативыне способы выделения текста нужны чтобы совмещать оба эти способа, например \_\___полужирный курсив__\_\_ внутри курсива*\*
## Списки
Для создания ненумерованного списка используются зведочки (*) или символы (+) с начала строки, например:
* 1 элемент
* 2 элемент
* 3 элемент

Для создания нумерованного списка следует использовать номера по порядку (1,2,3) с начала строки, например:
1. первый элемент
2. второй элемент
3. третий элемент
## Работа с изображениями
Чтобы вставить изображение в текст, нужно использовать следующую конструкцию \!\[Подсказка](https://url), например:
![изображение](https://drive.google.com/uc?export=view&id=1OOmJSmMMQ0Q0kPfH4G5iBDSg8SYgYm4H)
## Ссылки
Чтобы добавить ссылку, нужно использовать следующую конструкцию \[текст ссылки](https://url), например [Презентация урока](https://gbcdn.mrgcdn.ru/uploads/asset/3393303/attachment/40dac267015e43631f85b5caf42d364d.pdf)
## Работа с таблицам
Для создания таблиц используется следующая конструкция:
|\|Колонка 1\||Колонка 2\||Колонка 3\||
|-|-|-|
\|\-\|\-\|\-\|
|\|ячейка 1\||ячейка 2\||ячейка 3\||
|\|ячейка 1\||ячейка 2\||ячейка 3\||
## Цитаты
Для создания цитат используется символ >, например:
> цитата
## Заключение
#### Также на этом уроки мы рассмотрели новые команды **git**:
* git branch new_branch - создание новой ветки из текущей с именем new_branch
* git branch - отоброжает список все ветки и текущую
* git branch -d new_branch - удаление ветки с именем new_branch
* git log --graph - вывести лог веток в виде дерева
* git checkout new_branch - переключиться на ветку с именем new_branch
* git merge new_branch - слияние текущей ветки с веткой new_branch
